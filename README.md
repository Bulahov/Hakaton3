# AI-репетитор по техническим дисциплинам (RAG)

## Описание проекта

Данный проект представляет собой **MVP AI-репетитора по техническим дисциплинам**, разработанного в рамках хакатона при поддержке **Cloud.ru** — российского провайдера облачных сервисов и AI‑технологий.

AI‑репетитор построен на основе подхода **Retrieval-Augmented Generation (RAG)** и предназначен для:
- поиска информации по учебным материалам,
- формирования контекстных ответов на вопросы студентов,
- объяснения технических тем в понятной форме.

Вся логика проекта реализована на языке **Python** в формате **Jupyter Notebook**, что позволяет прозрачно продемонстрировать все этапы работы решения: от загрузки данных до генерации ответа пользователю.

## Цели и задачи проекта

Основная цель проекта — разработка **минимально жизнеспособного продукта (MVP)** AI‑репетитора, который:

- использует RAG‑pipeline для работы с учебными данными;
- осуществляет семантический поиск по векторным представлениям;
- генерирует ответы на основе найденного контекста;
- демонстрирует возможность масштабирования решения.

---

## Требования и ограничения

- Backend реализован на **Python**  
- Используется **RAG‑подход** (векторизация + поиск + LLM)  
- Код проекта размещён в репозитории  
- Все вычисления и демонстрация работы выполняются внутри Jupyter Notebook  
- Пользовательский интерфейс реализован в упрощённом виде (через ячейки ноутбука)

## Как запустить проект

Проект запускается локально через Jupyter Notebook.

### 1. Системные требования

- Python версии 3.x  
- Jupyter Notebook или Jupyter Lab  
- Операционная система: Windows / macOS / Linux  

### 2. Клонирование репозитория

```bash
git clone <repo_url>
cd <repo_name>
```

### 3. Запуск Jupyter Notebook

```bash
jupyter notebook
```

После запуска в браузере откроется интерфейс Jupyter.

---

### 4. Запуск основных скриптов (ячеек ноутбука)

Откройте файл ноутбука:

```
1_6_NeuroEmotions_hackathon3_AI_репетитор.ipynb
```

Далее **последовательно выполняйте ячейки сверху вниз**, так как каждая стадия зависит от результатов предыдущей.

На этом этапе выполняются:
- загрузка и предварительная обработка учебных данных;
- разбиение данных на смысловые чанки;
- подготовка текстов к векторизации.

⚠️ Пропуск ячеек может привести к некорректной работе последующих этапов.

---

### 5. Векторизация, поиск и генерация ответов

На финальных этапах ноутбука выполняются:

- построение векторных представлений (embeddings);
- сохранение и использование векторного хранилища;
- реализация семантического поиска по запросу пользователя;
- передача найденного контекста в языковую модель;
- генерация итогового ответа AI‑репетитора.

После выполнения всех ячеек пользователь может:
- задать вопрос по учебным материалам;
- получить сгенерированный ответ на основе релевантного контекста.

---

## Архитектура решения (логическая)

1. Загрузка и подготовка учебных данных  
2. Разбиение текста на чанки  
3. Векторизация текстов  
4. Поиск релевантного контекста  
5. Генерация ответа LLM  
6. Вывод результата пользователю
